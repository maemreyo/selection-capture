## 📊 Current State Analysis

### **What We Have (v0.1.1 + roadmap work-in-progress)**
✅ **Strong Foundation:**
- macOS-only implementation (solid Accessibility API integration)
- Synchronous, trait-based architecture
- Platform-neutral capture method model with automatic fallback
- Feature-gated Windows beta and Linux alpha scaffolds
- Optional async wrapper (`capture_async`) behind `async` feature
- Experimental backend-agnostic monitoring scaffold (`CaptureMonitor`)
- Professional OSS setup (docs, CI/CD, release automation)
- Published to crates.io with dual licensing

### **Current Limitations**
⚠️ **Platform Gap:**
- Windows beta backend is partially implemented (real clipboard read + active app detection),
  but UI Automation and IAccessible capture remain `Unavailable`
- Linux alpha backend remains scaffolded; AT-SPI/X11 paths are not yet real integrations
- No real AT-SPI/UIA/IAccessible capture on non-macOS yet
- Monitoring API has no built-in platform backend yet
- Limited extensibility for custom strategies and plugins

### **Market Opportunity**
Based on research and trends:

1. **Cross-platform demand**: Desktop automation tools are in high demand
2. **Accessibility improvements**: Linux accessibility stack is modernizing (AT-SPI 3.0)
3. **Clipboard monitoring**: Growing need for clipboard managers with smart features
4. **AI/ML integration**: Text capture as input for AI assistants
5. **Privacy-first**: Local processing, no cloud dependency

### **Phase 2: Async & Performance (Q4 2026)**
*Goal: Unblock async applications and improve performance*

#### **2.1 Async Support**
**Priority: HIGH** | **Effort: MEDIUM** | **Impact: HIGH**

**Why:**
- Modern Rust apps are async-first (Tokio, async-std)
- Blocking calls cause UX issues in GUI apps
- Enables integration with webview frameworks (Tauri, etc.)

**API Design:**
```rust
// Current sync API (stays unchanged)
pub fn capture(...) -> CaptureOutcome { ... }

// New async API (feature-gated)
#[cfg(feature = "async")]
pub async fn capture_async(...) -> CaptureOutcome { ... }

// Non-blocking variant
pub fn try_capture(...) -> Result<CaptureOutcome, WouldBlock> { ... }
```

**Implementation Options:**
1. **Tokio** - Most popular, but heavy dependency
2. **async-std** - Standard library approach
3. **futures-lite** - Minimal overhead
4. **Platform-native** - Use OS async primitives

**Recommendation:** Feature-gate with `tokio` as default, but keep core dependency-free.

**Status update (2026-03-29):** A minimal optional async wrapper is now available behind the
`async` feature. `capture_async(...)` delegates to the existing sync engine and avoids Tokio
runtime panics by only using `block_in_place` on a multi-thread runtime, falling back to direct
sync delegation otherwise.

**Verification commands:**
```bash
cargo test --features async --test async_capture --verbose
cargo test --features async --lib --verbose
```

### **A. Windows UI Automation Strategy**

```rust
// Detailed implementation plan for Windows

use windows::{
    Win32::UI::Accessibility::{
        IUIAutomation,
        TreeScope,
        Condition,
    },
    core::Interface,
};

pub struct WindowsPlatform {
    automation: IUIAutomation,
    cleanup_state: Mutex<CleanupState>,
}

impl WindowsPlatform {
    fn new() -> windows::Result<Self> {
        let automation = unsafe { 
            CoCreateInstance(&CUIAutomation8)? 
        };
        
        Ok(Self {
            automation,
            cleanup_state: Mutex::new(CleanupState::Clean),
        })
    }
    
    fn attempt_ui_automation(&self) -> PlatformAttemptResult {
        // 1. Get foreground window
        let hwnd = unsafe { GetForegroundWindow() };
        
        // 2. Create element from hwnd
        let element = unsafe {
            self.automation.ElementFromHandle(hwnd)
        }.map_err(|_| PlatformAttemptResult::PermissionDenied)?;
        
        // 3. Find focused element
        let focused = unsafe {
            self.automation.GetFocusedElement()
        };
        
        // 4. Get value pattern (for text)
        let value_pattern = focused
            .cast::<IUIAutomationValuePattern>()
            .ok();
            
        if let Some(pattern) = value_pattern {
            let text = unsafe {
                pattern.GetCurrentValue()
            }.to_string();
            
            if !text.is_empty() {
                return PlatformAttemptResult::Success(text);
            }
        }
        
        // 5. Fallback: TextRange pattern
        // ... (implementation details)
        
        PlatformAttemptResult::EmptySelection
    }
}
```

### **B. Linux AT-SPI Strategy**

```rust
// Detailed implementation plan for Linux

use atspi::{
    AccessibilityConnection,
    events::Event,
    proxy::accessible::AccessibleProxy,
};

pub struct LinuxPlatform {
    connection: AccessibilityConnection,
}

impl LinuxPlatform {
    fn new() -> Result<Self, atspi::Error> {
        let conn = AccessibilityConnection::default();
        conn.register_event("focus:")?;
        conn.register_event("object:text-selection-changed")?;
        
        Ok(Self {
            connection: conn,
        })
    }
    
    fn attempt_at_spi(&self) -> PlatformAttemptResult {
        // 1. Get desktop
        let desktop = self.connection.get_desktop(0);
        
        // 2. Get focused element
        let focused = desktop
            .get_accessible_at_point((0, 0))
            .or_else(|_| desktop.get_focused_object());
        
        // 3. Query text interface
        if let Ok(text) = focused.query_interface::<Text>() {
            let selection = text.get_selection(0);
            
            if !selection.is_empty() {
                return PlatformAttemptResult::Success(selection);
            }
        }
        
        // 4. Fallback: X11 selection
        // ... (implementation details)
        
        PlatformAttemptResult::EmptySelection
    }
}
```

### **C. Async Architecture Design**

```rust
// Proposed async API design

#[cfg(feature = "async")]
pub mod r#async {
    use tokio::sync::mpsc;
    
    pub struct AsyncCaptureEngine {
        tx: mpsc::Sender<CaptureRequest>,
        rx: mpsc::Receiver<CaptureResult>,
    }
    
    impl AsyncCaptureEngine {
        pub fn new() -> Self {
            let (tx, rx) = mpsc::channel(32);
            
            // Spawn background worker thread
            std::thread::spawn(move || {
                // Process requests asynchronously
            });
            
            Self { tx, rx }
        }
        
        pub async fn capture(&mut self) -> CaptureOutcome {
            self.tx.send(CaptureRequest::Capture).await?;
            self.rx.recv().await.ok_or(CaptureError::ChannelClosed)?
        }
        
        pub async fn start_monitoring<F>(
            &mut self,
            mut handler: F,
        ) -> Result<(), CaptureError>
        where
            F: FnMut(String) + Send + 'static,
        {
            self.tx.send(CaptureRequest::StartMonitoring).await?;
            
            while let Some(result) = self.rx.recv().await {
                match result {
                    CaptureResult::TextChanged(text) => handler(text),
                    CaptureResult::Error(e) => return Err(e),
                }
            }
            
            Ok(())
        }
    }
}
```
